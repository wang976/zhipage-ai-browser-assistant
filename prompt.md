你是一个资深 Chrome Extension 工程师。请基于 Chrome 浏览器扩展，开发一个“智页浏览器 AI 助手”，产品交互参考豆包浏览器 AI 助手，但不是完全复刻，而是在其基础上按本文档要求实现。

========================
一、项目目标
========================
实现一个 Chrome 插件形式的浏览器 AI 助手，名称为“智页浏览器 AI 助手”。

核心能力包括：
1. 主对话侧边栏
2. 多会话历史管理
3. 页面侧边悬浮头像入口
4. 网页划词工具栏
5. 模型配置与切换
6. 设置页
7. 快捷键配置

请直接生成一个可运行的完整项目，而不是只给设计思路。

========================
二、技术要求（硬性约束）
========================
1. 使用 Chrome Extension Manifest V3。
2. 推荐技术栈：
   - TypeScript
   - React
   - Vite
   - Tailwind CSS（可选，但建议用于快速实现 UI）
3. 必须包含以下扩展模块：
   - background/service worker
   - content scripts
   - side panel 页面
   - options/settings 页面
4. 数据持久化优先使用：

5. 主对话界面优先使用 Chrome Side Panel API 实现。
6. 代码结构必须清晰模块化，避免把所有逻辑写在一个文件里。
7. 需要提供 README，说明如何：
   - 安装依赖
   - 本地构建
   - 加载到 Chrome 扩展
   - 配置模型 API

3. 每个会话至少包含：

E. 头像侧边栏功能
--------------------------------
参考图：
- ./images/AvatarSidebar_1.png
- ./images/AvatarSidebar_2.png
- ./images/setting_ AvatarSidebar.png

功能说明：
1. 在普通网页中，页面右侧悬浮一个头像图标。
2. 鼠标悬停图标时，弹出操作菜单。
3. Ctrl+K 与点击悬浮头像，都应打开主对话侧边栏。
4. 悬浮菜单中只需要实现以下后三项：
   - 截图识别文字（预留接口）
   - 总结此页面（真实实现）
   - 翻译此页面（预留接口）

实现要求：
1. “总结此页面”点击后：
   - 打开主对话侧边栏
   - 自动向当前会话发送一条请求
   - 请求内容不应只包含 URL，还应尽量包含：
     - 当前页面标题
     - 当前页面 URL
     - 当前页面提取到的正文文本（做长度截断）
   - 例如可以组装为：
     “请总结此页面的核心内容，并给出结构化摘要。页面标题：... 页面URL：... 页面正文：...”
2. “截图识别文字”和“翻译此页面”先只保留菜单项与空实现入口：
   - 点击后可提示“暂未实现”
   - 代码中保留后续扩展点
3. 设置页中需要实现：
   - 是否始终显示头像侧边栏开关
   - 禁用网站列表管理
4. 对于禁用网站列表中的站点，不显示悬浮头像。

禁用网站匹配建议：
- 按 hostname 或 match pattern 存储
- 至少支持新增、删除、展示列表

当用户在网页中选中一段文字时，出现划词工具栏。

2. AI搜索（本期可不实现）
3. 解释
4. 朗读（本期可不实现）
5. 翻译
6. chat

本期必须实现以下功能：

1）头像
- 点击后仅打开主对话侧边栏
- 不做额外处理

2）解释
- 点击后，将“选中文本 + 提示词‘解释一下’”发送给当前选中的模型
- 在页面中显示一个新的解释卡片
- 卡片右上角有关闭按钮
- 卡片在关闭前持续存在
- 卡片显示模型回复内容
- 风格尽量参考 explain_1 / explain_2

- 卡片顶部需要有：
  - 源语言：自动检测
  - 目标语言：下拉可选
- 默认目标语言可设为“中文简体”
- 卡片右上角可关闭
- 卡片在关闭前持续存在
- 风格尽量参考 translate.png

- 点击后，原划词工具栏切换成输入框形态，参考 chat_1.png
- 用户输入内容并点击发送后：
  - 打开主对话侧边栏
  - 把“用户输入提示词 + 选中文本”一起发送到当前会话
- 交互流程参考：
  - ./images/chat_2.png
  - ./images/chat_3.png

行为要求：
1. 划词工具栏是否启用、使用哪种外观，受设置页控制。
2. 至少实现两种外观样式切换，参考 setting_WordSelectionToolbar.png。
3. 只在用户选择了非空文本时出现。
4. 工具栏定位要尽量贴近选中文本区域。
5. 点击页面空白处后，可隐藏工具栏。
6. 解释/翻译卡片关闭前持续存在。
